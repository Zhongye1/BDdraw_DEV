以下是一份专业的分析思路框架，以及让 AI 生成标准技术文档的 Prompt（提示词）。

### 第一部分：前端功能模块分析框架（5D 分析法）

在分析任何模块时，建议按照以下 5 个维度（Dimension）进行拆解：

1. 表现层 (UI & Interaction)
   组件拆分：这个模块由哪些子组件构成？（原子组件 vs 容器组件）
   UI 库依赖：使用了什么组件库（AntD, Shadcn UI）？有哪些自定义样式（Tailwind, CSS Modules）？
   交互逻辑：用户点击/输入后，界面发生了什么变化（Loading, Modal 弹出, 路由跳转）？
2. 状态层 (State Management)
   本地状态：组件内部使用了哪些 useState / useReducer？
   全局状态：是否依赖 Redux / Zustand / Context？（例如：用户信息、权限控制）
   服务端状态：是否使用了 React Query / SWR 处理缓存和同步？
3. 数据层 (Data & Network)
   接口定义：调用了哪些 API？请求参数和返回结构是什么？
   类型定义：TypeScript 的 Interface/Type 是如何定义的？
   数据转换：前端是否对后端数据做了格式化（Adapter 模式）？
4. 逻辑层 (Business Logic)
   核心算法：有没有复杂的表单校验、权限判断、数据递归处理（如树形控件）？
   副作用：useEffect 里做了什么？（事件监听、Socket 连接、埋点上报）。
   Hooks 封装：是否有复用的 Custom Hooks？
5. 工程层 (Performance & Quality)
   性能优化：是否使用了 useMemo, React.memo, 懒加载 (lazy)？
   异常处理：接口报错怎么处理？空状态（Empty State）怎么展示？
   边界情况：弱网、长文本溢出、移动端适配。

### 第二部分：标准技术文档模板

一个合规的企业级前端文档应包含以下章节：
模块概述：一句话解释这个模块是干嘛的。
流程图/时序图：核心业务流程（推荐 Mermaid 语法）。
组件架构：文件目录树与组件依赖关系。
状态与数据流：State 结构定义与数据流向。
核心代码解析：关键函数的逻辑说明。
API 接口映射：关联的后端接口。
注意事项/TODO：已知的 Bug 或待优化项。

### 第三部分：AI 提示词 (Prompt) 指南

你可以将代码文件内容复制，配合以下提示词发给 ChatGPT/Claude/DeepSeek。
场景一：已有代码，生成详细说明文档

Prompt:

# Role

你是一位拥有 10 年经验的高级前端架构师和技术文档专家。你擅长编写清晰、规范、易读的代码说明文档。

# Task

请根据我提供的【代码片段/文件内容】，对该功能模块进行深度分析，并生成一份完整的技术实现文档。

# Input Code

(在此处粘贴你的代码，如果是多个文件，请标注文件名)

# Output Requirements (文档结构)

请严格按照以下 Markdown 格式输出：

## 1. 模块概述

- 简述该模块的功能目标和业务场景。

## 2. 架构设计

- **组件结构图**：使用 Mermaid 或文本树形结构展示组件层级。
- **技术栈关键点**：列出关键的库（如 React Hook Form, Zustand, Tailwind 等）及其用途。

## 3. 核心逻辑分析 (重点)

- **状态管理**：分析核心 State 的设计（本地 vs 全局），并解释其作用。
- **交互流程**：详细描述用户操作到页面响应的完整链路。
- **关键函数**：提取代码中复杂的函数（如数据处理、校验逻辑），用自然语言解释其算法或逻辑。

## 4. 数据交互

- 列出涉及的 API 请求。
- 展示核心 TypeScript 类型定义（Interface）。

## 5. 流程图 (Mermaid)

- 请生成一个 Mermaid `sequenceDiagram` 或 `flowchart`，描述核心业务的执行流程。

## 6. 边界与异常处理

- 代码中是如何处理 Loading、Error 或空数据的？
- 还有哪些潜在的优化空间？

# Style

- 语言风格：专业、客观、简洁。
- 格式：Markdown，代码块请注明语言。
  场景二：没有代码，只有需求，要求 AI 设计实现方案
  Prompt:
  code
  Markdown

# Role

你是一名精通 React、Tailwind CSS 和 TypeScript 的前端技术专家。

# Task

我需要开发一个【具体功能模块，例如：带有拖拽排序功能的文件上传列表】。请为我设计技术实现方案。

# Requirements

1. **技术栈**：Vite + React + TS + Tailwind + (推荐一个合适的第三方库，如 dnd-kit)。
2. **功能点**：
   - [功能 1]
   - [功能 2]
3. **输出内容**：
   - **目录结构**：建议的文件组织方式。
   - **核心状态设计**：需要维护哪些 State。
   - **关键代码骨架**：给出核心组件的伪代码或关键逻辑实现。
   - **难点攻克**：预判开发中可能遇到的坑（如性能、兼容性）并给出解决方案。
     💡 实际操作技巧
     分批投喂：如果模块代码很多（超过 5 个文件），不要一次性全扔给 AI。先发核心组件，再发 Utils 和 Hooks，最后要求它“基于以上所有文件生成总结文档”。
     要求 Mermaid：一定要让 AI 生成 Mermaid 代码，你可以直接在 Notion、GitHub 或 VSCode (Markdown Preview Mermaid Support) 中渲染成漂亮的流程图，这会让文档瞬间变得非常专业。
     关注“为什么”：如果 AI 解释了代码，你可以追问：“为什么要用 useCallback 包裹这个函数？”或者“这里为什么用 Zustand 而不是 Context？”，这有助于你深入理解原理，而不仅仅是复制文档。
